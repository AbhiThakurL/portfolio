10 Steps to Create a Django
Design in E-Commerce Website

Category:  Technology
Date:  Jan 2024
Author:  Paul Trueman

Creating an e-commerce website in Python involves multiple steps, from setting up the back-end to building the front-end and managing databases. Below, I’ll walk you through the basic steps for building an e-commerce site using the Django framework, which is one of the most popular frameworks in Python for web development.

"Creativity is intelligence having fun." - Abhishek Thakur
Set Up Your Environment

Install Python : Make sure Python is installed on your machine.
Install Django: Run the following command to install Django.

"pip install django"
Start a New Django Project

In your terminal, navigate to the directory where you want to create the project, then run:
This will create a Django project named my_ecommerce and an app called shop.

django-admin startproject my_ecommerce
cd my_ecommerce
python manage.py startapp shop
Set Up Your Database

Django comes with SQLite by default, but you can switch to PostgreSQL or MySQL if needed.

" DATABASES =
{
'default':
{
'ENGINE': 'django.db.backends.sqlite3',
# Change to 'django.db.backends.postgresql' for PostgreSQL
'NAME': BASE_DIR / 'db.sqlite3',
}
} "
Define Models for Your E-commerce

In shop/models.py, define your models. You can start by creating basic models like Product, Category, and Order.

from django.db import models

class Category(models.Model):
 name = models.CharField(max_length=255)
 slug = models.SlugField(unique=True)
 def __str__(self):
 return self.name
class Product(models.Model):
 name = models.CharField(max_length=255)
 category = models.ForeignKey(Category, on_delete=models.CASCADE)
 description = models.TextField()
 price = models.DecimalField(max_digits=10, decimal_places=2)
 stock = models.IntegerField()
 available = models.BooleanField(default=True)
 slug = models.SlugField(unique=True)
 image = models.ImageField(upload_to='products/', blank=True, null=True)

 def __str__(self):
  return self.name
Create and Run Migrations

After defining the models, apply the migrations to create the database tables.

python manage.py makemigrations.
python manage.py migrate
Create Admin Panel

To manage products, categories, and orders easily, register your models in the admin interface.In shop/admin.py:

from django.contrib import admin
from .models import Product, Category
@admin.register(Product)
class ProductAdmin(admin.ModelAdmin):
 list_display = ('name', 'category', 'price', 'stock', 'available')
  prepopulated_fields = {'slug': ('name',)}

@admin.register(Category)
class CategoryAdmin(admin.ModelAdmin):
  list_display = ('name',)
  prepopulated_fields = {'slug': ('name',)}

Now you can access the Django admin by creating a superuser:

python manage.py createsuperuser
Build Views and Templates

In shop/views.py, define views to display product listings, product details, and handle the shopping cart. Example of a product listing view:

from django.shortcuts import render, get_object_or_404
from .models import Product, Category

def product_list(request, category_slug=None):
category = None
categories = Category.objects.all()
products = Product.objects.filter(available=True)

if category_slug:
category = get_object_or_404(Category, slug=category_slug)
products = products.filter(category=category)

return render(request, 'shop/product_list.html', {
'category': category,
'categories': categories,
'products': products
})

In shop/templates/shop/product_list.html, create the template:

<h1>Products</h1>
<ul>
{% for product in products %}
<li>{{ product.name }} - {{ product.price }}</li>
{% endfor %}
</ul>
Handle URLs

In shop/urls.py, define the routes for product listings and product details:

from django.urls import path
from . import views

urlpatterns = [
path('', views.product_list, name='product_list'),
path('<slug:category_slug>/', views.product_list, name='product_list_by_category'),
]

Don’t forget to include shop.urls in your main urls.py:

from django.urls import include, path

urlpatterns = [
path('admin/', admin.site.urls),
path('shop/', include('shop.urls')),
]
Shopping Cart and Checkout

You can extend this further by implementing a shopping cart system. Here’s a simple structure: Cart model: Keep track of cart items. Checkout view: Process orders. Payments: Use libraries like Stripe or PayPal for payment integration. Example of a shopping cart model:

class Cart(models.Model):
product = models.ForeignKey(Product, on_delete=models.CASCADE)
quantity = models.IntegerField()

def get_total_price(self):
return self.product.price * self.quantity
Deploy Your E-commerce Website

Finally, you’ll need to deploy your website. You can deploy Django applications using platforms like: Heroku DigitalOcean AWS Make sure you: Set up Gunicorn or another WSGI server. Use Nginx as a reverse proxy. Optional Features: User Authentication: Allow users to create accounts, view their order history, etc. Search Functionality: Add a search bar for users to find products. Product Reviews: Enable product reviews and ratings. Email Notifications: Send order confirmations to users.
